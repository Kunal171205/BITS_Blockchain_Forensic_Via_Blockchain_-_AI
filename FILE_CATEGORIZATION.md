# Comprehensive Project File Categorization

This is a complete breakdown of all files and directories in the **Blockchain Document Forensics & AI Forgery Detection** project after the legacy code cleanup.

## 🚀 1. Core Runtime (Essential)
These files are required for the system to function.

| Path | Description |
| :--- | :--- |
| **[api_server.py](api_server.py)** | The main Flask API server. Bridges the frontend, ML, and Blockchain. |
| **[blockchain/blockchain/](blockchain/blockchain/)** | The Smart Contract environment (Hardhat). |
| ├── [contracts/](blockchain/blockchain/contracts/) | Solidity source code (`DocumentRegistry.sol`). |
| └── [artifacts/](blockchain/blockchain/artifacts/) | Compiled ABI and Bytecode for the blockchain. |
| **[business_logic/](business_logic/)** | The brain of the app: ML inference and verification logic. |
| **[models/](models/)** | Trained ML weight files (e.g., `unet_tamper_model.pkl`). **Do not delete.** |
| **[frontend/](frontend/)** | The Next.js web application (React, Tailwind, ethers.js). |
| **[.env](.env)** | Configuration secrets (Contract address, Private keys, RPC URL). |

---

## 🛠️ 2. Development & Setup Tools
Used for building, deploying, or troubleshooting.

| Path | Description |
| :--- | :--- |
| **[deploy_local.py](deploy_local.py)** | Script to deploy contracts and update `.env` automatically. |
| **[RUN_GUIDE.md](../RUN_GUIDE.md)** | Step-by-step instructions for running the project. |
| **[scripts/](scripts/)** | Helper scripts for blockchain tasks. |
| **[debug_model.py](debug_model.py)** | Quick script to verify model loading and integrity. |
| **[extract_images.py](extract_images.py)** | Tool for pre-processing or extracting data from images. |

---

## 🧪 3. Research & Data Testing
Files used for development testing or data research.

- **Setup Guides**: `RUN_GUIDE.md`, `README.md`.
- **Test Scripts**: All `test_*.py`, `debug_model.py`, `extract_images.py`.
- **Research**: All `notebook*` files and `.ipynb` notebooks.
- **Sample Data**: All `dummy*.jpg` and `out*.json` files.
- **Temporary**: `temp/` folder, `temp_mask.npy`, `output.txt`.

---

## 📦 4. Registry & Persistence
Where the app stores its local records.

| Path | Description |
| :--- | :--- |
| **[admin_registry.json](admin_registry.json)** | Local database of documents uploaded by the admin. |
| **[temp/](temp/)** | Temporary storage for active uploads and Ganache state. |
| **temp_mask.npy** | Cache of the latest forgery mask generated by the AI. |
